# ADR-001: Отказ от использования BIND9 в качестве основного DNS-резолвера

## Статус
Отклонено

## Дата
2026-03-20

## Контекст
На этапе выбора базового стэка проекта на первой неделе рассматривались различные варианты DNS-серверов. Исторически BIND9 является де-факто стандартом и самым популярным DNS-демоном в Unix-системах. Однако специфика задач требовала сервера, ориентированного в первую очередь на гибкое кэширование, безопасность (DNSSEC-валидатор) и производительность рекурсивных запросов без избыточного функционала авторитетного сервера.

## Решение
Мы приняли решение отказаться от использования BIND9 в качестве основного резолвера и вместо этого выбрать Unbound (см. ADR-002).

## Обоснование
- **Специализация:** BIND9 исторически проектировался как мощный авторитетный сервер, к которому со временем добавились функции кэширования. Unbound, напротив, представляет собой специализированный кэширующий рекурсивный DNS-резолвер. Он более легковесный и показывает лучшую эффективность при работе с кэшами.
- **Инструментарий работы с кэшем:** Во второй неделе проекта потребовалось манипулировать кэшем и симулировать вытеснение (eviction). Интерфейс `unbound-control` предоставляет более удобные команды для дампа (`dump_cache`) и контроля расширенной статистики попаданий/промахов по сравнению с `rndc` из BIND9.
- **Поддержка DNSSEC и локальных вмешательств:** Unbound изначально проектировался с фокусом на DNSSEC и имеет встроенную директиву `domain-insecure`, которая идеально подошла для обхода валидации "подмененных" (spoofed) доменов (w4). В нативном BIND9 подобные трюки со смешанным состоянием зон требовали бы более сложной архитектуры views (представлений).

## Последствия
- Поскольку Unbound не поддерживает полноценную оркестрацию первичных/вторичных (Primary/Secondary) зон с трансферами AXFR/IXFR, мы лишаемся возможности масштабировать архитектуру до классического авторитетного нейм-кластера. Для наших задач это не является критичным.
- Работа с авторитетными заглушками реализована через директивы `local-zone`/`local-data`, а не через полноценные zone file (хотя частичная поддержка файлов формата RFC 1035 присутствует для инъекций).
