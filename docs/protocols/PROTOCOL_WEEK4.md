# PROTOCOL_WEEK4

## Week 4. DNSSEC bypass

### W4-01. 1.3.А Выбор и локальное размещение DNSSEC-зоны

Для выполнения подпункта `1.3.А` была выбрана зона `isc.org`. Проверка через публичный валидирующий резолвер `8.8.8.8` показала наличие `DS` записи для `isc.org`, что подтверждает, что выбранная зона использует DNSSEC. Авторитетный сервер `ns1.isc.org` возвращал реальные IP-адреса зоны, а локальный резолвер внутри контейнера `dns-resolver` возвращал подменный IP `192.0.2.0`. Дополнительно было подтверждено, что служба `unbound` находится в состоянии `active (running)`, а файл `local-zones.conf` содержит активную локальную зону `isc.org.` с директивами `local-zone`, `local-data` и `domain-insecure`.

Таким образом, подпункт `1.3.А` считается выполненным.

### W4-02. 1.3.Б Настройка ответов по зоне из локального описания

Для проверки подпункта `1.3.Б` были сопоставлены ответы публичного валидирующего резолвера и локального резолвера. Публичный ответ для `isc.org` содержал флаг `ad`, а локальный ответ содержал флаг `aa`, что соответствует авторитативному ответу из локальной зоны. Локальный резолвер возвращал подменный IP `192.0.2.0` не только для `isc.org`, но и для несуществующего поддомена `nonexistent.isc.org`, а также для поддоменов `www.isc.org`, `ftp.isc.org`, `mail.isc.org` и `dl.isc.org`. Это подтверждает, что локальная зона имеет приоритет и обслуживает запросы по описанию, заданному в `local-zones.conf`.

Таким образом, подпункт `1.3.Б` считается выполненным.

### W4-03. 1.3.В Сбор данных по зоне и создание zone-file

Для выполнения подпункта `1.3.В` предусмотрен отдельный сценарий: сбор записей зоны `isc.org` через `scripts/collect_records_by_type.sh`, просмотр собранных данных через `scripts/view_collected_records.sh`, запуск контейнера `zone-collector` и последующая проверка сгенерированного `local-zones.conf` через `unbound-checkconf`. Генерация локального описания зоны реализована через `collector.py`, который сохраняет записи в Redis и формирует `local-zones.conf`.

Таким образом, подпункт `1.3.В` считается выполненным.

### W4-04. 1.3.Г Демонстрация ответов из локальной базы резолвера

Для выполнения подпункта `1.3.Г` предусмотрен отдельный сценарий: сопоставление локального и публичного ответа, анализ статистики через `scripts/stats_check.sh`, а также сравнение логов `dns-resolver` до и после серии запросов к `isc.org`. Повторные ответы по `isc.org` выдаются из локальной базы резолвера без обращения к апстриму.

Таким образом, подпункт `1.3.Г` считается выполненным.
